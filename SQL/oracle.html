Oracle

Oracle主目录用户

Oracle数据库在安装的时候会要求指定一个非管理员的普通系统账户作为主目录用户，用来安装和配置Oracle主目录，运行Oracle主目录的服务。
在Linux环境中，必须切换到主目录用户才能调用sqlplus命令（大概是因为我的环境是将Oracle安装给主目录用户，而不是所有用户），在Windows环境则没有这个限制。

根目录和主目录

Oracle根目录（或者说基目录 ORACLE_BASE）是Oracle产品的顶级目录，所有产品或者是不同版本的数据库软件都会安装到这里。
主目录（ORACLE_HOME）是每一个Oracle产品自身的安装目录，该目录为Oracle根目录的子孙目录。

在Oracle database 12c（Windows环境）中，根目录是C:\app\oracle，主目录是C:\app\oracle\product\12.2.0\dbhome_1

SQL*Plus

SQL*Plus是Oracle数据库最基本的管理工具，它可以执行SQL*Plus的内置命令、SQL语句、PL/SQL代码块、外部shell的命令。
就像是MySQL登陆后可以执行help等非SQL语句一样。
sqlplus命令可以用来启动SQL*Plus，或者是启动SQL*Plus并连接到特定数据库。

SQL*Plus环境配置

$ORACLE_HOME/sqlplus/admin/glogin.sql，这个文件类似于bash的 .bash_profile，在用户登录的时候被执行。
添加这一行将sqlplus的提示符修改为 user@instance 这种形式。
set sqlprompt "_user'@'_connect_identifier SQL> "

数据库与实例

数据库就是硬盘上存储的数据库相关的文件的集合；实例是操作数据库的进程、内存空间的集合。
一个实例一生只能装载一个数据库，一个数据库可以被多个实例装载。
目前我遇到的情况都是一个数据库对应一个实例，所以对我而言，数据库和实例这两个词是可以互换的。
12c版本中的新特性CDB与PDB可以实现一个实例对应多个数据库的情况，详见网友博客。

用户、权限、角色

角色就是多个权限的集合，可以直接给用户授权，或者是把角色赋予给用户。
系统默认提供三个数据库管理相关的角色——connect、resource、dba。

  1. Connect 角色，是授予最终用户的典型权利，最基本的权利，能够连接到ORACLE数据库中，并在对其他用户的表有访问权限时，做SELECT、UPDATE、INSERTT等操作。
  2. Resource 角色，是授予开发人员的，能在自己的方案中创建表、序列、视图等。
  3. DBA角色，是授予数据库管理员的。

另外还有两个拥有系统权限的角色——sysdba、sysoper。

sysdba与sysoper拥有数据库系统管理的权限，比如打开/关闭/创建/删除/备份/恢复数据库。
sysdba的权限比sysoper还要高，且只有sys账户拥有sysdba角色，任何账户以sysdba角色登陆后都会自动转成sys账户。
登陆时不指定系统管理角色，就是以normal角色登陆。

sysdba、sysoper与connect、resource、dba并不冲突，也没交集。
用户以sysdba角色登陆是为了管理数据库，如果该用户同时具备dba权限，那他还可以操作数据库中的数据。

system账户和sys账户：

sys账户拥有dba，sysdba，sysoper角色，是数据库实例的最高权限账户。
system拥有dba角色，且只能使用normal身份登录。
system账户我一般用不到，sys账户的话，一般使用它的sysdba身份登录来管理数据库。

表空间与用户

表空间是一种逻辑上的概念，数据实际存储再数据文件中，多个数据文件可以组织成一个表空间。
一个表空间只能属于一个数据库，一个表空间可以供多个用户使用，一个用户也可以使用多个表空间。

如果不手动指定用户的默认表空间和临时表空间，用户就会使用数据库的默认表空间和默认的临时表空间。
临时表空间主要用途是在数据库进行排序运算、管理索引、访问视图等操作时提供临时的运算空间，当运算完成之后系统会自动清理。

默认的用户表空间是USERS，默认的临时表空间是TEMP 。

可以在 dba_free_space 和 dba_data_files 这两张表格中查到表空间的总大小和剩余大小。
select tablespace_name,sum(BYTES)/(1024*1024) as "ts_size（M）" from dba_data_files group by tablespace_name;

查看表空间的数据文件的路径。
select file_name from dba_data_files where tablespace_name='USERS';

修改表空间的大小可以修改其中一个数据文件的大小，或者给表空间添加/删除数据文件。
alter database datafile 'F:\ORACLE\PRODUCT\10.2.0\ORADATA\ORCL\xxx.DBF' resize 1024m;

模式与用户

模式与用户是一一对应的关系，模式名默认就是用户名，模式是存放数据库对象的地方。
模式不能被单独创建，要创建模式必须创建用户。
用户不仅能访问自己模式中的数据库对象，也能访问其他用户的模式中的数据库对象。
模式是逻辑数据结构，并不是实际的数据存储文件。

oracle和MySQL的【用户与数据库的关系】的区别

oracle是用户属于数据库，多个数据库之间是独立的，每个数据库都有自己的管理员账户，虽然名字都是sys和system这两个。用户可以访问自己的模式下的数据库对象，也可以授予其他用户访问自己模式下的数据库对象的权限。dba角色的用户可以访问该数据库中任意模式中的数据库对象。

MySQL中的用户属于默认的名为mysql的数据库。root用户是MySQL的管理员，管理所有的数据库。仅有root用户可以创建其他用户，并把操作数据库或表格的权限授予给他。
