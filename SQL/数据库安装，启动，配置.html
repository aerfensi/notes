mysql默认会有一个名为mysql的数据库，里面的user表格就保存了用户的数据。
所以对用户的操作就是操作这张表格，比如要查看有多少用户。

SELECT user FROM mysql.user;

MySQL创建用户

host：指定该用户在哪个主机上可以登陆，如果是本地用户可用。 localhost，如果想让该用户可以从任意远程主机登陆，可以使用通配符 %。
CREATE USER 'username'@'host' IDENTIFIED BY 'password';

MySQL为用户授权

privileges：用户的操作权限，如SELECT，INSERT，UPDATE等，如果要授予所的权限则使用ALL。
如果希望把所有数据库和表格的权限都授予，可以使用通配符*.*。
GRANT privileges ON databasename.tablename TO 'username'@'host';

MySQL修改/设置用户密码

SET PASSWORD FOR 'username'@'host' = PASSWORD('newpassword');
如果需要修改当前用户的密码。
SET PASSWORD = PASSWORD("newpassword");

MySQL撤销用户权限

REVOKE privilege ON databasename.tablename FROM 'username'@'host';

MySQL删除用户

DROP USER 'username'@'host';

Oracle

监听服务

当用户登录数据库时，用户进程发送请求，服务器端的监听程序监听到用户进程的请求，使用listener.ora核对用户连接信息，如果正确，则启动一个服务器进程，然后把用户进程直接或者间接交付给服务器进程。
上面讲的是专用服务器模式，对于共享服务器连接模式，则是将监听到的用户请求转发调度器，再由调试器调度共享服务器进程处理。
如果是访问本地的数据库实例，不需要启动监听服务，仅需要启动OracleService<instance_name>。比如实例名为orcl，则这个服务是OracleServiceORCL。

lsnrctl

该命令用来管理 Oracle 的监听服务（前提是该服务不能被禁用），用的最多的就是 start、stop、status。
该命令应该是 listener control 的简写。

数据库的启动与关闭

默认情况下数据库是随着数据库服务一同启动的，Windows可以在注册表中更改，HKEY_LOCAL_MACHINE\SOFTWARE\ORACLE\KEY_OraDb10g_home1\ORA_ORCL_AUTOSTART

数据库有四种状态：shutdown、nomount（实例启动）、mount（打开控制文件）、open（所有文件都按照控制文件的描述打开）。
在nomount状态，也有部分视图已经打开了，比如v$instance。
startup [open/mout/nomount]：默认是open
shutdown [normal/transactional/immediate/abort]：默认是normal，shutdown后，不能让有新用户连接，但是对已有的连接的处理方式因选项而异。
normal：等待其他用户主动断开。
transactional：等待正在处理的事务完成后，断开用户连接。
immediate：打断正在处理的事务并回滚，断开用户连接。
abort：直接打断正在处理的事务，断开用户连接。

select status from v$instance; 可以查看当前实例的状态，分为started、mounted、open。
实例一旦被启动，如果希望改变其状态，不能直接使用startup命令，必须先shutdown。 或者手动改变其状态，alter database mount/open;

Oracle创建用户与授权

xxxxxxxxxx
-- 创建用户
create user username identified by password;
​
-- 将某个表格的某个权限赋予给某个用户
grant privilege on tablename to username;
​
-- 将某个角色赋予给某个用户，可以仅赋予dba，或者赋予connect和resource，仅赋予resource的话，用户没有连接数据库的权限的
grant connect,resource,dba to username;
​
-- 取消权限
revoke privilege on tablename from username;
​
-- 取消角色
revoke connect from username;

用户锁定

用户有三种状态open、locked、expired（过期）。
用户多次输错密码会导致locked，密码期限到后会导致expired。
user_users表格中有账户的锁定的信息，dba_users表格中则包含其他用户的锁定信息。
select username,account_status,lock_date from user_users;
alter user username account unlock/lock; 解锁或锁定账户。
到期的账户再次登录的时候会要求更改密码，更改之后就可以正常登录了。
