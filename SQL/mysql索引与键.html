什么是索引？

索引保存在表格外面，用来加快表格的读取速度，同时也会减慢表格的修改速度
一般为经常出现在WHERE语句或者ORDER BY语句中的列创建索引
网上对索引的分类说法不一，我不清楚索引总共能分成几类，但是不同的类不一定是互斥的。比如一个唯一索引同时也可以是一个单列索引（或者是复合索引）

索引与键的关系

主键、外键、唯一键在创建时必然会自动创建一个索引，称之为唯一索引、主键索引、外键索引

唯一索引与唯一键

  • Mysql唯一索引 唯一约束

唯一键是一种约束，保证该列中的数据的唯一性，但是有两个例外
NULL值不受该约束，同列中可以有多个NULL
空白符都被认为是长度为0的字符串，比如' '与''是同等的，所以唯一索引的列中只能有一个全为空白符的数据

一张表格中可以有多个唯一键

唯一键需要保证数据的唯一性就必须在数据被插入时查询整列，为了加快查询速度就需要创建索引
所以唯一键与唯一索引虽然是不同的概念，但是可以被视作同一个东西
因为创建唯一键的时候一定会自动创建唯一索引，唯一索引被删除的时候，唯一键也就被删除了

主键索引与主键

主键是在唯一键的基础上添加下面两个约束：
一张表格只能有一个主键
主键列中不能有NULL

强烈建议使用自增列（AUTO_INCREMENT）作为主键

主键的创建于删除同唯一键，把UNIQUE改成PRIMAEY KEY即可

外键与外键索引

在表A中创建一个外键列，将其指向表格B的主键列。表A就是子表，表B就是父表
外键能联系两张表格，让两张表格能完成一些联动操作

  1. MySQL中，只有InnoDB引擎支持外键，故两张表格都必须是InnoDB引擎的
  2. 外键列与其指向的主键列必须有相似的数据类型，比如都是数字
  3. 外键列同样会自动创建索引
  4. 同一张表格可以有多个外键
  5. 同一个主键列可以同时作为多张表格的外键的引用

无法在创建列的时候添加foreign key，只能在创建表格时，或者对已存在的列添加foreign key

外键约束的作用

  1. 外键列中的数据必须能在对应的主键列中找到相同的值
  2. 外键列中的数据可以有重复的
  3. 外键列允许存在NULL，虽然主键列中是不可能有NULL的

默认情况，子表中某一行的外键列的数据与父表中的主键列的某一个数据相同时，父表不能删除那一行，也不能修改那一行的主键
除非子表在设置外键时定义了相关操作

  1. cascade：父表删除或更新时自动删除或更新子表匹配的行
  2. set null：父表删除或更新时，子表外键列自动设置为null【要用该选项，外键列不能为not null】
  3. restrict：父表不能更新或删除
  4. no action（创建外键时的默认操作就是这个）：在mysql中，这个同restrict

Alter Table tbl Add [ Constraint foreign_key_name ] Foreign Key (col) References tbl2(col) ON DELETE CASCADE ON UPDATE CASCADE

索引名字

当手动指定索引/键的名字时，规范为——"表名字段1名字段2名"

对于primary key由于一张表格只能有一个，删除时也无需指定键名，它的名字其实可以不关心
而其他的，创建索引时如果没有指定名字，删除索引时就需要去查查索引的名字了
