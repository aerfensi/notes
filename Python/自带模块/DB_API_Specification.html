Python DB API Specification

Python Database API Specification规定了Python访问数据库的API，不管是Python内置的还是第三方的数据库模块都是这个API规范的实现。

Python访问数据库的简单流程为：创建connection→获取cursor→执行sql语句→关闭cursor→关闭connection

connection

数据库连接对象，用来提交事务与回滚。

database：一个path-like object，path-like object就是能代表一个路径的对象，简单的就是使用绝对/相对路径的字符串。

check_same_thread：创建的connection对象只能被当前线程使用，除非该参数传入False。

cursor

游标，用来执行sql语句，以及获取查询结果。pymysql的cursor可以执行命令，比如用use db切换数据库，sqlite3和cx_Oracle则不行，但是不要这么做，cursor还是指用来执行sql语句的好。
执行的sql语句不要用分号结尾！！！

connection的cursor()方法可以得到一个cursor对象。

  • cursor.execute(sql[, parameters])： 执行一条SQL语句
  • cursor.executemany(sql, seq_of_parameters)：同样只能有一条SQL语句，但是他能多次执行该语句

使用select查询数据后，可以使用如下方法获取数据：

  1. fetchone() -> tuple：获取一行数据
  2. fetchmany([size=cursor.arraysize]) -> list ：获取n行数据，数量由传入的size决定，不传入则由cursor的arraysize属性决定。用位置参数的方式传入size参数，而不要用关键字参数，因为不同的数据库模块这个size的变量名不一定相同。
  3. fetchall() -> list：list中的每一个元素都是tuple
  4. rowcount：cursor对象的rowcount成员中记录了查询结果的行数

cursor是一个迭代器，所以直接for循环也是可以的。 如果执行的是修改语句，那最后一定要使用connection的commit()方法提交事务；如果cursor执行sql语句时报错，则需要使用connection的rollback()方法回退事务。
