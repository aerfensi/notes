发送请求

import requests
headers={
    'User-Agent': 'Mozilla/5.0 (X11; Linux x86_64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/64.0.3282.167 Safari/537.36'
}
# params是get请求是发送的数据，也可以不用params参数，而把网址修改为

response=requests.get('http://httpbin.org/get', params={'name': 'Tom'}, headers=headers, timeout=5, verify=False)

请求发送完毕后，requests的连接也就关闭了，除非将get函数的stream参数设置为True
此时需要等待请求数据被全部读取或者手动调用response.close()才能关闭
好在response支是上下文管理器

hook

可以给请求挂上钩子来操控请求的过程，比如

xxxxxxxxxx
def callback(r,**kwargs):
    pass
​
requests.get('https://www.baidu.com',hooks={'response':callback})

hooks字典的key是有规定的，response指的是在请求结束后回调（其实我目前只见过response，不知道其他还有啥）
response指定的回调函数接收的第一个参数是请求结果response对象，**kwargs是请求中传入的关键字参数，比如timeout（其实就是请求中传入的**kwargs）
requests的请求会自动添加一些关键字参数，所以**kwargs永不为空
官方文档中的回调函数的参数列表是r, *args, **kwargs，我发现*args是空的，没有参数传进来

响应

  • response.encoding : str : 当前响应体使用的编码，改变它的话，当访问响应体时将使用新的编码
  • response.text : str : 以文本的形式访问响应体
  • response.content : bytes : 以字节形式访问响应体，text等效于response.content.decode(response.encoding)
  • response.iter_content(chunk_size)：如果响应的内容太大，比如下载文件时。chunk_size的单位是bytes。使用该方法必须保证requests的get函数中的stream参数为True
  • response.iter_lines(chunk_size)
  • response.status_code : int : 状态码，除了判断它是否是200外，还可以用requests.codes.ok来判断，这个requests.codes里面定义了所有的状态码
  • response.ok : bool : 并不能代表请求的网页被顺利打开，因为网址被重定向后，这个ok也会是True
  • response.reason : str : 返回状态的原因，如果是200，这个就是‘OK’
  • response.headers : requests.structures.CaseInsensitiveDict : 是一种字典，因为响应头中的数据也是键值对
  • response.url : str : 请求的网址
  • response.history : list : 重定向历史
  • response.cookies : requests.cookies.RequestsCookieJar
  • response.json() ： 响应内容是json的话，根据它得到dict对象

下载二进制文件

xxxxxxxxxx
import requests
​
headers={
    'User-Agent': 'Mozilla/5.0 (X11; Linux x86_64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/64.0.3282.167 Safari/537.36'
}
response=requests.get('https://upload-images.jianshu.io/upload_images/5901754-2e680e38f1042190.jpg', headers=headers, timeout=5)
# 如果仅仅需要简单的下载的话，直接用二进制打开一个文件，把content写进去就可以了
with open('test.jpg', 'wb') as picture:
    picture.write(response.content)

网站重定向

有些时候输入一个网址后会调到另一个网址，比如输入http://baidu.com会自动跳转到http://www.baidu.com，这些跳转的历史会保存在response.history这个list中

xxxxxxxxxx
import requests
headers={
    'User-Agent': 'Mozilla/5.0 (X11; Linux x86_64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/64.0.3282.167 Safari/537.36'
}
response=requests.get('http://baidu.com', headers=headers, timeout=5)
​
print(response.url)
print(response.history)
print(response.history[0].url)
​
# 输出
# https://www.baidu.com/
# [<Response [302]>]
# http://baidu.com/

如果要禁止跳转，可以在请求中加参数
只能禁止永久跳转（响应状态码301），但是不能禁止暂时跳转（响应状态码302）

xxxxxxxxxx
r = requests.get('http://github.com', allow_redirects=False)

SSL证书验证

验证服务器

requests自带火狐CA证书库，对于一些常见的https网站，可以直接认证通过的

xxxxxxxxxx
from requests.packages import urllib3  
import requests  
​
# 关闭警告
urllib3.disable_warnings()
# 不进行证书校验
response = requests.get('https://www.12306.cn', verify=False)
​
# 设置本地证书  
response = requests.get('https://www.12306.cn', verify='/path/crt')

异常

所有异常都继承自requests.exceptions.RequestException
异常中包含了request和response，但是这两个数据可能是None，继承自RequestException的异常类中除了__doc__外啥都没写，所以要查询异常的信息的话就用

xxxxxxxxxx
except RequestException as e:
    print(e.__doc__)

# x-www-form-urlencoded
# 设置header的content-type是application/x-www-form-urlencoded
data ='name=%E7%8B%97%E7%8B%97'.encode('utf-8')

from urllib.parse import urlencode
data={
    'name':'Tom',
    'password':'Admin@123'
}
r=requests.post("http://httpbin.org/post", data=urlencode(data))
​
# json
# 设置header的content-type为application/json
# data参数支持的对象类型为 Dictionary, list of tuples, bytes, or file-like object
# 不支持字符串，所以使用字符串的话，需要编码成bytes
data='{"j":"json"}'.encode('utf-8')
# 或者直接使用dict
data={'j':'json'}
r = requests.post("http://httpbin.org/post", data=data)
#还有一种方式是使用json参数，它支持str和dict类型的对象
r = requests.post("http://httpbin.org/post", json='{"j":"json"}')
​
# xml
# 手动指定content-type为application/xml或者是text/xml
data='<xml>xml</xml>'.encode('utf-8')
r = requests.post("http://httpbin.org/post", data=data)
​
# multipart/form-data
# 不要手动设置header的content-type为multipart/form-data，要让requests自动添加。因为没有办法手动在headers和body中同时设置boundary，只能靠requests自动生成。
# filename中不要有斜杠
files = {
    "field1" : ("filename1", open("filePath1", "rb")),
    "field2" : ("filename2", open("filePath2", "rb"), "image/jpeg"),
    "field3" : ("filename3", open("filePath3", "rb"), "image/jpeg", {"refer" :"localhost"})
}
# files的格式为 files = {'name': (<filename>, <file object>,<content type>, <custom_headers>)}
# 对应http消息体
# Content-Disposition: form-data; name=’name’;filename=<filename>
# Content-Type: <content type>
# <file object>
# --boundary
​
# 如果上传的是文本的话<filename>可以直接为None，然后在<file object>中传入一个字符串即可
r = requests.post("http://httpbin.org/post", files=files)

cookies

response的headers是Set-Cookie字段告诉浏览器，哪些数据要写在cookies中，下次请求的时候带上
最简单的方式就是用浏览器登录，然后查看浏览器的request中的cookies，复制下来就行了。

使用session可以在下次请求时自动带上cookie。

xxxxxxxxxx
import requests
​
headers = {
    # 也可以使用get方法的cookies关键字参数，不过那样需要构建一个RequestsCookieJar对象比较麻烦
    'Cookie': r'xxxxxxxxxx',
    'User-Agent': r'xxxxxxxxxxxx'
}
​
response = requests.get('https://www.zhihu.com/', headers=headers)

代理设置

设置普通代理

xxxxxxxxxx
import requests  
  
proxies = {  
    "http": "http://127.0.0.1:9743",  
    "https": "https://127.0.0.1:9743",  
}  
# 往请求中设置代理(proxies)  
response = requests.get("https://www.taobao.com", proxies=proxies)  

设置带有用户名和密码的代理

xxxxxxxxxx
import requests  
  
proxies = {  
    "http": "http://user:password@127.0.0.1:9743/",  
}  
response = requests.get("https://www.taobao.com", proxies=proxies)
