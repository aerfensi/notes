hook

可以给请求挂上钩子来操控请求的过程，比如

​x
1
def callback(r,**kwargs):
2
    pass
3
​
4
requests.get('https://www.baidu.com',hooks={'response':callback})

hooks字典的key是有规定的，response指的是在请求结束后回调（其实我目前只见过response，不知道其他还有啥）
response指定的回调函数接收的第一个参数是请求结果response对象，**kwargs是请求中传入的关键字参数，比如timeout（其实就是请求中传入的**kwargs）
requests的请求会自动添加一些关键字参数，所以**kwargs永不为空
官方文档中的回调函数的参数列表是r, *args, **kwargs，我发现*args是空的，没有参数传进来

响应

  • response.encoding : str : 当前响应体使用的编码，改变它的话，当访问响应体时将使用新的编码
  • response.text : str : 以文本的形式访问响应体
  • response.content : bytes : 以字节形式访问响应体，text等效于response.content.decode(response.encoding)
  • response.iter_content(chunk_size)：如果响应的内容太大，比如下载文件时。chunk_size的单位是bytes。使用该方法必须保证requests的get函数中的stream参数为True
  • response.iter_lines(chunk_size)
  • response.status_code : int : 状态码，除了判断它是否是200外，还可以用requests.codes.ok来判断，这个requests.codes里面定义了所有的状态码
  • response.ok : bool : 并不能代表请求的网页被顺利打开，因为网址被重定向后，这个ok也会是True
  • response.reason : str : 返回状态的原因，如果是200，这个就是‘OK’
  • response.headers : requests.structures.CaseInsensitiveDict : 是一种字典，因为响应头中的数据也是键值对
  • response.url : str : 请求的网址
  • response.history : list : 重定向历史
  • response.cookies : requests.cookies.RequestsCookieJar
  • response.json() ： 响应内容是json的话，根据它得到dict对象
