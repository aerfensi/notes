coding=utf-8

指明当前 python 文件的编码，如果不指明，python3 默认当前脚本文件是 utf-8，Windows 上也这样。

docstring

在模块、类、方法、函数中定义的第一个语句，如果该语句是字符串的话，该字符串就是 docstring。
网上也称其为多行注释，其实它既是注释，也是 python 的文档。
docstring 可以是三双引号、三单引号、也可以是双引号（此时只能是单行字符串，不能是多行了）。
docstring 会被赋值到模块、类、方法、函数的__doc__属性中，如果没有 docstring，__doc__就是 None。

docstring的格式有很多种，比如：

import

import aa 和 from aa import * 的区别：

1 是导入模块，用 globals() 函数可以看到一个模块对象
2 是导入模块中的内容，用 globals() 函数看到的是模块中的内容

比如from logging import *后，可以直接使用error()函数，而不需要loggging.error()。

模块中的魔法成员

__doc__：模块的docstring。
__name__：当前模块的名字，如果当前模块是被执行的模块，则他的值为__main__。
__file__：当前模块的路径。如果执行当前脚本是使用绝对路径，如python /xxx/xxx.py，则它的值为绝对路径；同理，它的值也可能是相对路径。所以要得到当前脚本的所在目录，可使用os.path.dirname(os.path.abspath(__file__))。

查看模块中的有哪些成员

globals()：得到一个当前模块所有成员的变量名和变量值的字典。
sys.modules[__name__].__dict__：效果同上，sys.modules是包含所有已导入的模块的模块名和模块对象的 dict，可以用它获得当前模块的对象，然后使用其__dict__属性。

工程结构

每个 python 包中都需要有一个__init__.py文件，该文件可以将一个普通目录标记为 python 的包。
如果直接指定要导入的模块，比如import package.module或者from package import module，那__init__.py可以是一个空文件，甚至不要这个文件也没关系。
如果希望批量导入模块，比如from package import *，则必须要有__init__.py，且该文件中必须定义__all__属性。

启动 python

双击运行的话，如果报错，cmd 窗口就直接退出了。所以我一般使用python script.py来运行。
有的时候也需要使用python -m script.py。-m 参数在文档中的解释是把“将 python 模块作为脚本来运行”。不是很懂，目前我知道的有两点：

1、加入 -m 参数后，python 会去安装目录的 Lib 目录中查找是否有需要被执行的 python 模块。如果不加的话，python pydoc -h，这种就提示找不到文件。python -m pydoc -h这样就可以，注意不要加 .py 后缀名。
2、 不加 -m，会将脚本所在的目录加入到 sys.path 中；加 -m，会将当前 cmd 路径加入到 sys.path 中。sys.path 中保存了 python 导入模块的搜索路径，所以 -m 参数会影响到 python 脚本能否正确地 import 需要的模块。

命名语法规则

类/异常：大驼峰
包/模块/变量/函数/方法：下划线
全局/类中的变量或者常量：全大写，下划线

Python中可以适当使用匈牙利命名法，因为动态语言不知道变量类型，匈牙利命名法可以改善代码可读性。

变量/函数/方法何时需要加下划线？。
并不是说每两个单词之间都要加上下划线，我看Google代码中也是有的加，有的不加，我不知道Google的规则。
我目前先遵循动名词之间加，名词之间不加的原则吧。比如add_item和filename。
变量名最好控制在3个单词之内，但是函数/方法/模块成员的名字可以长一些。
