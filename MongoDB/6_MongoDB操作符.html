Field Update Operators

$unset：删除document中的一个field，用法是{$unset:{fieldName:""}}，没错就是这么奇怪
$set：如果需要更新的field不存在，则创建该field
$setOnInsert：只有因为upsert:true导致新的document将被插入时，才会去插入field和value，否则无影响

数组相关： $：表示数组中的第一个元素{ $set: { "grades.$" : 82 } }
$[]：表示所有数组元素{ $inc: { "grades.$[]": 10 } }，所有元素都加10
$addToSet：给数组追加元素，数组field不存在则创建，将被追加的元素已存在于数组则不追加 { $addToSet: { tags: "accessories" } }
$pop：删除数组中的第一个/最后一个元素，{ $pop: { scores: -1 } } 删除scores数组中的第一个元素
$pull：删除数组中所有符合条件的元素，{ $pull: { votes: { $gte: 6 } } }，删除votes数组中所有大于6的元素
$pullAll：删除与指定数组中的元素相同的元素，{ $pullAll: { scores: [ 0, 5 ] } }，删除scores中所有值为0、5的元素

$[<identifier>]：与<option>中的arrayFilters连用，指代数组中所有符合arrayFilters的元素

aggregation pipeline operator

与其他操作符的辨析

comparison query operator

包含除了$nin（not in）之外的所有comparison query operator，但是用法不同。
{key:{$eq:1}} —— comparison query operator —— key的值是否等于1 {$eq:['$field1','$field2']} —— aggregation pipeline operator —— field1与field2的值是否相等

Logical Query Operators

支持$nor以外的其他三个Logical Query Operators（$and、$or、$not），用法与Logical Query Operators相同
{ $or: [ <expression1>, <expression2>, ... ] }

Field Update Operators

这两种操作符中，有些操作符同名但意义不同；有些则名字相似，意义也相似

Field Update Operators
{ $inc: { <field1>: <amount1>,...} 给某个字段增加数值
{ $mul: { <field1>: <amount1>,...}

aggregation pipeline operator
{ $add: [ <expression1>, <expression2>, ... ] } 把两个expression相加，expression可以是字段名或者具体的数值
{ $multiply: [ <expression1>, <expression2>, ... ] }

Field Update Operators
{ $min: { <field1>: <value1>, ... } } 指定的数据比原字段的数据小，则更新，否则不更新
{ $max: { <field1>: <value1>, ... } }

aggregation pipeline operator
{ $min: <expression> } 取出同组中同字段名的最小值
{ $min: [ <expression1>, <expression2> ... ] } 取出同一个document中多个字段中的最小值 $max同上
