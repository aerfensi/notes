MongoDB文档

数据类型

如果说JSON是JavaScript对象的字符串表示，那BSON可以理解为扩展的JSON的二进制表示
它是以二进制的方式存储，但是读取的时候还是要转化成当前语言支持的对象
比如JSON文档就是一个纯文本文档，但是Python读取它后会转化成Python支持的list，dict等数据类型

MongoDB是用BSON文档来存储数据的，所以只能存储BSON支持的数据类型
MongoDB的驱动程序可以把BSON的数据类型转化成各种变成语言支持的数据类型，使这些语言能够访问BSON数据
Mongo shell就是JavaScript的shell，为了让JavaScript支持BSON中数据类型，Mongo shell中添加了一些BSON支持的数据类型对应的JavaScript对象——Data Types in the mongo Shell

mongo shell 数值类型

JavaScript对于所有数字，不管整数小数都视为number类型，或者是其包装类型Number。而BSON对于数字是有int、long、double、decimal（128位浮点型）之分的。Mongo shell将所有JavaScript中的数字视为double，除非手动new出BSON支持的数字类型对用的js对象，比如decimal就是NumberDecimal
就和前面说的一样，编程语言支持的数据类型和BSON支持的数据类型会互相转化

_id

所有document都必须有有个_id字段，该字段用来确保当前collection中document的唯一性，也就是说一个collection中的所有document的_id都必须是不同的
_id可以是任意数据类型

_id类似于mysql中的主键，但是有如下区别
不手动指定_id，MongoDB会自动生成，mysql据说不创建主键也会自动生成一个隐藏的主键，反正我没见过……
_id是可以为null的（当然，多个document的_id不能同时为null），而mysql的主键是不能为null的

MongoDB没有自增约束，网上也有说不要设置mysql自增主键这种东西，MongoDB的id不用自增

插入

如果collection不存在，则自动创建collection

  • db.collection.insert(<document or array of documents> [,option])：插入一条或多条document
  • db.collection.insertOne(obj [,option])：option是Write Concern相关选项，详见这部分笔记
  • db.collection.insertMany(array [,option])：array是元素为BSON的数组

默认情况下传入array以插入多个document的情况是ordered的，如果array中有一个document插入出现错误，后面的document也不会被插入
把option中的ordered字段设置为false{ordered:false}，可以把插入变为非ordered

除了$set外，还有其他的Update Operators

建议使用mongodb官方教程中的更新方法，上面的方法虽然也可以，但是比较老了

  • db.collection.updateOne(<filter>, <update>, <options>)
  • db.collection.updateMany(<filter>, <update>, <options>)
  • db.collection.replaceOne(<filter>, <replacement>, <options>)

upsert:true插入的document

<query>与<update>中的具体字段都会被插入到这个新的document中
比如db.test.update({ myArray:[1,2] }, { $min: { "key":2} },{upsert:true})
如果没有查到document，则会插入一个新的document，该document包括field myArray:[1,2]，key:2
因为没有查到document，所以就没有名为“key”的field，此时不管是使用$set/$inc/$min/$max，都是直接给key赋值
如果<query>中的是{myArray:{$size:3}}这种，则不会有名为myArray的field被插入

replaceOne

replaceOne用来代替save方法
save是指定已存在的id则替换，否则插入
replaceOne是替换query找到的第一个元素，如果没找到且option中的upsert:true，则插入新元素

replaceOne的replacement中不需要指定id，如果指定了id，则必须与被替换的document的id相同
replacement中不要加Field update operator

$or操作符后面跟一个数组，只要数组中的某一个条件为真，该语句就为真
$lt是比较操作符，/^p/是正则表达式
可以看到要使用比较操作符，需要把操作符和数据值用花括号包围

所以该条件语句翻译成JavaScript为if(status === "A" && (qty<30 || item.match(/"^p"/)))

db.inventory.find( { status: { $in: [ "A", "D" ] } } )——status为"A"或者是"D"

限制查询结果中的字段

db.inventory.find( { status: "A" }, { item: 1, status: 1 } )：查询结果会显示_id，item，status三个字段
没错，1就表示要显示该字段，但是_id就算不设置，也是会显示的，为了隐藏它可以把它设为0——{ item: 1, status: 1, _id:0}
查询结果中字段的显示顺序是document插入时的顺序，也就是说<projection>中的field即使交换顺序也不会有影响，这和SQL不同
db.inventory.find( { status: "A" }, { status: 0, instock: 0 } )：如果<projection>中只有排除的字段，则表示除了这些字段，其他的都显示
