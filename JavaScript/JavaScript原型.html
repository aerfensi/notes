对于JavaScript的原型链，我上网翻看一些博客后知道了如下几点。

  1. 所有函数都有prototype成员，但只需关心作为构造函数的函数的prototype
  2. prototype是一个Object，称为当前函数的原型，也叫做原型对象
  3. 所有对象都有一个__proto__成员，指向其构造函数的prototype对象
  4. 无法在对象中找到的成员，JS引擎会顺着__proto__，去prototype中找
  5. this绑定的成员，只存在当前对象中，prototype绑定的成员可被所有由该构造函数创建的对象共享

但总是感觉对这个概念理解的不太清晰，我希望通过这篇文章能够理清一下原型链的知识。

我会加入一些自己的猜测，因为我实在是不想为了理解原型链去查大量的资料。
我这里说的【对象】都是指一般理解的对象，JS中函数也是对象，我称之为【函数对象】，以作区分。

JS的对象从哪里来？

学习一门面向对象的语言的对象相关知识，最重要的是知道对象是从哪里来的。我先回顾一下Java和python的对象。

Java的所有类都是Object的子类，即便是匿名内部类也是如此，所以所有对象都是Object的实例。故Java所有的对象都最终来自于Object。
python的对象也是由类来创建的，不同的是类也是对象，而类对象是type类型的，type类自身也是type类型的，所以python的所有对象都最终来自于type对象。

JS没有类，所有对象是用构造函数创建的，即使是使用{}，也是隐式调用了构造函数。JS的Object是一个构造函数，我发现对象的原型链最终会追溯到Object.prototype，而Object.prototype.__proto__ == null，也就是说Object.prototype是原型链的顶端。

总结一下，所有对象都由Object构造函数创建的，即使使用new constructor()，也需要调用到Object构造函数，同时Object的原型也是所有对象原型链的顶端。

这是误导我的地方，我以此认为JS和Java一样，所有对象都来自于Object。
其实不是这样的，因为如果认为所有对象都来自于Object，还有一个问题无法解释，就是——函数也是对象，函数从哪里来？

new 做了什么？

在说明函数是从哪里来之前，先解决一个问题——如果我用自定义的构造函数创建对象时也会调用Object构造函数的话，那new操作到底做了哪些事？

简单的说new constructor()，做了三件事，用代码表示就是

2
var obj  = {};
3
//把新之创建的对象的 __proto__ 指向构造函数的prototype
4

5

6
Func.call(obj);

函数对象从哪里来？

首先，函数才有prototype，但是所有对象（包括函数对象）都有__proto__，既然它指向对象构造函数的原型，那我就用它来找函数对象的构造函数。
这里用String函数来举例。
String.__proto__ == Function.prototype; //true
可以看到String函数对象是由Function函数构造的，所以猜测Function函数是所有函数对象的构造器，可以试一下如下代码 

xxxxxxxxxx
4
1
var func=new Function();
2
typeof func; //"function"
3
var obj=new Object();
4
typeof obj; //"object"

可以看到new Function()可以创建函数对象，对比new Object()可以创建普通对象，所以Function是所有函数对象的构造器。
既然Object也是函数，那Object自然也是由Function创建的，Object.__proto__ == Function.prototype; //true。
而Function对象的构造函数就是它本身，Function.__proto__ == Function.prototype。

总结一下，JS中函数创建对象，Function创建函数对象，故所有对象都最终来自于Function。其实对比Java也能想通，原则就是谁创建对象，对象就从谁那里来。

要接受所有对象都来自Function，需要先明白一件事——原型链不是继承。网上常常把JS的原型链说成是JS实现对象继承的方式，这很容易让我觉得所有对象都继承自Object，所以所有对象都最终来自Object。

其实应该说所有普通对象都由Object创建，且他们的原型链的顶部就是Object.prototype，这个理解起来就没什么困难。
有问题的是函数对象的原型链，函数对象都由Function创建，但是原型链的顶端确是Object.prototype。

因为__proto__说到底是一个属性，，可以改的，我认为Function.prototype.__proto__是被改成了指向Object.prototype，目的是为了让函数对象也能有普通对象的行为变现。

总之原型链不是继承，所以什么是原型链？在此之前我先了解一下，什么是原型。

原型是什么？

编程中有一种设计模式，称之为【原型模式】。原型模式的作用是复制对象本身以创建更多的对象。
但是JS中的并不会复制原型，我看到的说法是，JS中通过原型链，让原型链下游的对象可以访问上游对象的属性，达到类似原型复制的效果（新创建的对象也有与原型相同的属性）

Object.prototype是所有对象原型链的顶端，它必须是一个object，这样，所有对象才能共有一些属性。
所有函数对象的原型都会追溯到Function.prototype，它必须是一个function，这样所有函数对象才能共有一些属性。typeof Function.prototype; //"function"。
当然函数对象的原型链最终也会到达Object.prototype。

对象创建的顺序

我认为JS中的函数与对象大概是这么来的

  1. Function创建了Function.prototype
  2. Function.__proto__指向Function.prototype
  3. Function创建Object函数
  4. Object函数创建Object.prototype
  5. Object.prototype.__proto__指向null（注意，null是一个字面值，不是对象。typeof null得到object，这是个历史遗留问题）
  6. Function.prototype.__proto__指向Object.prototype
  7. 所有函数对象由Function创建；所有普通对象由Object创建

原型链不是继承

我先回顾一下Java和python的继承。

Java：子类继承自父类，子类的实例可以访问子类的成员，也可以访问父类的成员，各个实例的成员相互独立（除静态变量外）。思路清晰，实例从继承链条（我暂且称类之间的因继承关系形成的链条为继承链条）上来，继承链条的上游也是实例的上游。

python：python中类也是对象，和Java有很大的区别，但也能类比Java中的继承。类同样作为创建对象__new__()，初始化对象__init__()的模板，子类实例对象也能访问父类的成员。（是能访问父类类对象的成员，不是父类实例对象的成员）

说回JS，JS没有类！继承是对于类而言的，没有类，哪里来的继承？JS中的原型链其实是委托，也就是对象自己不实现某个功能，而是调用外部对象的功能。

constructor

每个prototype都有一个constructor属性，指向其所在的构造函数（Func.prototype.constructor == Func）。对象会根据原型链检索可用的属性，也就是说obj.constructor就是它的构造函数。

用这个属性可以轻松创建同类型的对象。
用constructor属性来查看对象从那里来的也很方便，比如 

xxxxxxxxxx
2
1

2
Function.constructor; //Function()

Object的构造器是Function，而Function的构造器是自身

总结

  1. 对象由函数创建，函数对象由Function创建，所以对象最终来源于Function
  2. Function创建函数，Object创建对象，所以说是Function创建了Object，用它来创建对象（包括Objct.prototype）
  3. 大多数情况下，原型链可以反映对象创建的链条。我不是说原型链等同对象创建的链条，而是说这两者有对应关系
  4. 有些时候__proto__并不指向对象的构造器的prototype，比如Function.prototype.__proto__ == Object.prototype
