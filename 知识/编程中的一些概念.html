编程中的一些概念

线程安全

线程安全就是多条线程访问同一段代码时，不会产生不确定的结果，也就是说多次执行结果应该是一样的。依靠线程 同步来保证线程安全，Java中使用同步锁(Synchronized)来实现线程同步。

回调

网上对于回调的说法很乱，而且举的例子常常给我一种错觉，以为回调是A在调用B的同时，B反过来调用A，比如网上的例子常常是这样的：
A对象调用B对象的方法C，同时将自己作为参数传给方法C，然后方法C再调用A的方法B。
我认为这种理解并不准确。
回调的文字表述是——“don't call me, I'll call you back”，更生活化的理解是“不要问我这件事做完了没，我做完了会告诉你的”。

我觉得回调的流程应该是： A告诉B在某个时刻调用A。到了这个约定的时刻，B调用A。
这里A和B调用的不一定是对方的方法，也可能是对方全局或局部变量的方法。
举个例子，比如Android中我们常常在Activity中为button设置监听器。

});

我现在假设这个Activity的名字就activity，来看这个流程：
activity调用button的setOnClickListener对象，告诉button我有一个局部的匿名内部类对象（也就是继承自OnClickListener的这个对象），你在被点击时记得调用它的OnClick方法。于是当button对象被点击时，这个onClick方法被调用了。
这样就可以理解为什么Android中onXXX方法都被叫做回调方法。
还有一个问题是类似与Activity生命周期的方法，Activity好像并没有调用哪个对象的方法，也没把自己传给哪个对象，好让它在适当的时机来调用自己的生命周期方法。

event hook

What is the difference between hook and callback?

来自stack overflow：
callback：回调，一个函数注册到一个API上，在适当的时机这个函数会被回调。
event hook：事件钩子，或者直接叫做hook，可以理解为更通用的回调，它可以修改API的调用，比如修改传递给API的参数。

callback是一种特殊的hook，这两者的区别比较模糊，在有些语言中甚至hook就是callback。

方法与函数

我以前一直以为方法和函数是同一种东西的不同叫法，就和句柄、引用，可是事实并不是如此。
不同语言对于函数与方法的区别是不同的，比如java只有方法，而c++只有函数，类中的函数被称为成员函数。
Python中方法与函数的区别是，绑定了非type类型的对象的就是方法，否则就是函数。

内存泄漏和内存溢出

内存泄漏，即部分对象虽然已经不再使用，但是因为有root持有引用，所以并没有被销毁，所占用的内存一直没有被释放。一次两次发生影响不大。如果频繁发生，那么可用内存会渐渐不足，最终在某一次请求内存时发现内存不足而发生oom（即Java的OutOfMemoryError）。这里要明确一个概念，只有强引用会发生内存泄漏，而weak等引用因为其特殊机制，所以影响不大。
详见Android内存泄漏学习笔记

两者的区别：
内存溢出 out of memory，是指程序在申请内存时，没有足够的内存空间供其使用，出现out of memory；比如申请了一个integer,但给它存了long才能存下的数，那就是内存溢出。
内存泄露 memory leak，是指程序在申请内存后，无法释放已申请的内存空间，一次内存泄露危害可以忽略，但内存泄露堆积后果很严重，无论多少内存,迟早会被占光。

行尾符

在打字机时代换行符（LF \r 换到下一行）和回车符（CR \n 光标移到行首 ）是两个不同的符号，但是到了计算机时代，回车这一个字符即可做到原先换行回车两个字符才能实现的效果。
Windows 沿用了打字机时代的做法，用 CRLF（\n\r）在行尾表示换行。 Linux 使用 LF（\n）在行尾表示换行。
但是由于大量的编程语言使用 \n 作为换行，所以其实在 Windows 中编程的时候，它会自动对 CRLF 和 LF 做转换。

UTF-8 与 BOM

Linux 的 UTF-8 不带 BOM，而 windows 的带。Windows 特有，所以还是使用不带 BOM 的编码，这样更通用一些。

这个 BOM 就是文件开头的一个特殊字符，用来表明当前使用的是哪一种 Unicode 编码（UTF-8，UTF-16BE，UTF-16LE）。Windows 与 Linux 在这方面的区别有很多原因，其中一个原因是微软对于兼容性的高要求，微软希望系统能兼容更多的字符编码（想想 Linux 经常出现中文乱码）。

二进制

说来惭愧，大学毕业连二进制都没搞懂。
我这里会用一些实际问题来描述二进制的概念。

二进制第一个是符号位，正数为0，负数为1。
正数补全是往左边添0，负数则是添1。
0001 → 0000 0001。
1001 → 1111 1001。

二进制的表示范围

八位二进制的话能表示 7~-8，我从大学开始就不清楚，为什么负数能多表示一个，对于正数而言，最大值是 0111；我当时想当然地以为负数的最小值是 1111（-7），确实如此，可是计算机中存储的是二进制的补码，不是原码。

正数的补码就是本身。
负数的补码是在原码的基础上对数值位取反后+1，符号位不变，所以 -7 在计算机中的表示是 1001，而 -1 的表示是 1111。
负数能多一个的原因是因为0只有一种表示形式，就是+0（0000），对于-0（1000）而言，对它取补码也变成了 1 0000，这是个四位二进制数，所以最左边的1被舍弃变成了 0000。

一个四位二进制，最多16种变化，因为0使用了正数的符号位，所以正数少一个变化。
多出来的-8，用哪个表示呢？用-1（1111）不断地-1，减到最后能发现最小的是1000，也就是-8，也就是-0的原码。或者另一种思路是用-8的原码（1 1000），求补码后再舍弃最左边的一位，也可以得到（1000）。

int溢出

最大的int值+1后会变成最小的int值，同理，最小的int值-1会变成最大的int值。
简单的说就是数值位能表示的数为【最大/最小】时，再对其【加上/减去】数值，使得符号位发生了【进位/借位】。

0111（7） + 0001 = 1000（-8）
1000（-8） - 0001 = 0111（7）

缓冲（buffer）与缓存（cache）

Cache和Buffer都是缓存，主要区别是什么？

buffer（缓冲）是为了减少短时间内发起的IO次数，如果有100的字节，分100次写入到文件，就要浪费大量的IO资源，这时可以用一个缓冲区来存储这些资源，每存到10个后，写入一次，这样总共就写了10次。
cache（缓存）位于速度相差较大的两种硬件之间，用于协调两者数据传输速度差异的结构。就是把高速设备需要频繁访问的数据放在cache中，数据先在cache中查找，没有的话，再去低速设备中查找。就是为了加快读取速度。

buffer是从内存中划分出来的空间，cache也可以是从内存中划分出来的，但是cache有专门的硬件，比如CPU都会带三级缓存，硬盘也有板载缓存，但是我没听说有什么专门的硬件是来支持buffer的。

表达式和语句

编程中常常听到这两个概念。
表达式（expression）是值、变量和运算符的组合，表达式一定会有返回值，表达式可以作为赋值语句右边的语句。
语句包含表达式，还有其他诸如if语句，while语句。

命名空间和作用域

命名空间就是保存有变量名字和变量值的一个容器，在 python 中是一个字典。python 的命名空间本身也属于更外层的一个命名空间，也就是说一个字典是另一个字典的元素，这样实现了命令空间的嵌套。

作用域是一个代码块，在这个代码块中，可以直接访问部分命名空间中的变量，比如 python 中可按照 L-E-G-B 的顺序访问命名空间中的变量。
所谓直接访问，就是直接使用变量名字，而不是【对象.变量】这种形式。
