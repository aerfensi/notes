CSS 选择器与 xpath 的区别

属性

CSS 把所有属性值当做字符串处理，只能做【等于】，【包含】这种操作。
xpath 可以对属性值做【大于】，【小于】这种数值判断，前提是 HTML 中的属性值没有引号包围；[attr>value]（xpath 表达式中的属性不需要 @，value 也不需要引号了）。

多值属性

CSS 可以很方便地对多值属性，比如用 .class 时，只需要 class 属性的某一个值符合选择器就会被选中，或者是使用~=比较符。
xpath 仅仅是将多值属性当做是一个【中间带有空格的字符串】，无法直接定位包含某个值的属性。

伪类与轴

CSS 的伪类和 xpath 的轴都是对节点的修饰，作用一样。
CSS 伪类：p:only-child，P 是它的父类的唯一子节点。
xpath轴：child::p，P 是某个节点的子节点，child 作为默认轴，可以不写，同时 html 作为最外面的一个根节点，居然也带有 child 的轴，好奇怪。

多重条件

xpath 可以很方便地写多重条件，使用 and、or 运算符；css 选择器好像没办法实现多重条件。

css伪元素

上述css选择器中用单冒号开头的是伪类，css3规定用双冒号开头的是伪元素。 但是有些网站为了兼容旧浏览器，伪元素也采用单冒号。 故要真正区分伪元素与伪类，只能靠名字。

  • ::text，取出文本节点
  • ::attr(href)，去除属性值，没错，属性名字上不用加引号

xpath

xpath选取包含某种元素的元素tag[tag] xpath对同一个元素的多重条件判断，p[position() = 1 and @class="name"] 这里表示元素位置不能用下标，一定要用函数 我觉得xpath的方括号[]有两种用法，1，下标取值；2，表达式运算

节点位置

CSS的节点位置用伪类来实现，比如 p:nth-child(3) xpath的节点位置可以用下标实现，/bookstore/book[1]

xpath常用轴

  • 祖先：//e/ancestor::*，e节点的所有祖先节点
  • 下面的兄弟节点：//e/following-sibling::*
  • 上面的兄弟节点：//e/preceding-sibling::*
  • 下面的兄弟节点和它们的子节点：//e/following::*
  • 当前节点及其祖先节点：//e/ancestor-or-self::*
  • 当前节点及其子孙节点：//e/descendant-or-self::*

xpath常用函数

下标来取位置，下标从0开始；position()函数确实从1开始计数的！

string()是将参数转为字符串，text()是取出当前节点的文本子节点。 在lxml中还有另外的用法，xpath不能这么用。

print(html.xpath('string()'))：取出所有文本节点，并把它们连成一个字符串。 print(html.xpath('//text()'))：取出所有文本子节点，并把它们组成一个list列表。

xpath只能取出元素节点，不能直接取出文本节点。 我这么说的依据是，chropath（一个chrome上的xpath扩展）不能用xpath取出文本节点；chrome的开发者工具不能复制出文本节点的xpath。
