setup和teardown

settings中的Test Setup/Teardown，将作为当前测试套件中所有测试用例的默认，也就是说测试用例不显示指定setup/teardown，就会使用settings中的设置。
settings中的Suite Setup/Teardown是属于测试套件的，只会在测试套件开始和结束时执行一次。
测试用例中的语法为[Setup/Teardown]，它会覆盖测试套件中的Test Setup/Teardown。如果希望测试用例不使用[Setup/Teardown]，也不使用测试套件中默认的Test Setup/Teardown，那就在测试用例中使用空的[Setup/Teardown]。
不管是哪种setup/teardown，语法都是一样的，后面只能接一个关键字+它的参数。

xxxxxxxxxx
5
1
*** Settings ***
2
Test Setup        Log                 'Test Setup'
3
Test Teardown     log                 'Test Teardown'
4
Suite Setup       Log                 'Suite Setup'
5
Suite Teardown    log                 'Suite Teardown'

Setup/Teardown中只能运行一个关键字，除非使用Run Keywords关键字

变量引用

${var}可以用在变量的定义中，也可以用在变量的引用中（${var}会被其实际的值替换）。

${var}引用变量时，语法为${<RF中定义的变量><python运算/python函数的调用/其他RF变量>}。

使用@{list}或&{dict}引用列表和字典时，列表或字典中的元素会被分散传入，比如： 

xxxxxxxxxx
9
1
*** Variables ***
2
@{list}     1    2    3    4
3
​
4
*** Test Case ***
5
untitled
6
    # Log将传入四个参数：1、2、3、4
7
    Log    @{list}
8
    # Log将传入一个参数：[1,2,3,4]
9
    Log    ${list}

同理：传参${dict}将传入一个字典对象，传参&{dict}将传入多个关键字参数，而@{dict}将传入多个位置参数（即dict的key）

变量的优先级为：

  1. RF内置变量（这些变量不会被覆盖，可以任意使用）
  2. 测试用例中使用被关键字返回值赋值的变量
  3. 测试套件中导入的变量文件
  4. 测试套件中定义的变量（*** Variables ***里边的）

条件语句

xxxxxxxxxx
3
1
Run Keyword If    ${var} is None    Log    True
2
...    ELSE    IF    ${var} == 1    Log    1
3
...    ELSE    Log    False

Run Keyword If时最常见的条件关键字，它会将判断条件中的变量替换为实际的值，然后将整个条件语句使用python的eval()函数运算，最后判断条件的真假。简单的说条件语句就是可以使用RF中的变量的python语句。
后面的ELSE IF和ELSE都不是关键字， 而是传递给Run Keyword If的参数。【...】这个省略号的作用是当整个关键字语句过长时，可以用它来换行。当然，将整个Run Keyword If语句写成一行也是可以的。
Run Keyword If的条件后面只能接一个关键字，如果希望符合某个条件后，执行多个关键字，那只能将这些关键字提前组合成新的关键字。

Keywords

关键字不区分大小写，而且使用空格代替python函数名中的下斜杠是可以的。像这种形式的写法都是合法的。 get length、Get Length、get_length、Get_length

关键字失败后，测试用例就会失败。我想着让关键字失败后，不标记测试用例失败，而是进入另一个分支，目前看来这个比较麻烦，而且似乎也不符合robot framework的设计。
一些常见的问题都会导致关键字失败，比如Wait Until Element Is Visible关键字可以设置timeout，但是一旦超时，直接标记关键字失败，而不会继续执行下面的流程；Get Text关键字可以通过locator获取文本，但是如果定位不到元素，直接标记失败，而不是返回None。

关键字的优先级顺序为：

  1. 测试套件文件中创建的用户关键字
  2. 外部资源文件中引入的关键字
  3. 外部库中引入的关键字→标准库中定义的关键字。

用【模块名.关键字】访问被覆盖的外层关键字。

注释和特殊字符

注释可以使用#，或者是关键字Comment，Comment 这是一条注释。
\可以对#等特殊字符做转义。

RF会忽略掉字段前后的空格，如果希望在字段前后加空格的话，可以使用\转义空格，或者是${SPACE}。这个的好处是需要连续多个空格时，不需要写成\ \ \,直接使用${SPACE*3}。
如果希望使用空字符串，则可以使用单一的字符\，最好还是使用${EMPTY}
