Intent

四大组件与intent各个属性的关系

以下讨论的属性包括action，category，data，type

  1. Activity：全都能用，category可能只有activity能用，所以其他组件谨慎使用category
  2. Service：全都能用，但是Google建议使用显示启动Service，所以action和category最好不用
  3. BroadcastReceiver：全都能用
  4. ContentProvider：不能用intent启动，所以全都不能用

ComponentName

intent对象设置component的方法：setComponent(ComponentName name)
ComponentName的构造器：ComponentName(Context pkg, Class<?> cls)，ComponentName(Context pkg, String cls)，ComponentName(String pkg, String cls)

Action

对应<intent-filter>元素中的<action>元素
intent对象设置action的方法：setAction(String action)
一个Intent只有一个action

标准 Activity Action

标准Broadcast Action（加粗表示只能由系统发出）

ACTION_POWER_CONNECTED和ACTION_BATTERY_CHANGED的区别是前者只在连上电源时发送一次，而后者会在充电过程中不断被发送，告知应用程序当前手机的电量等信息

Category

对应<intent-filter>元素中的<category>元素
intent对象添加category的方法：addCategory(String category)
intent对象删除category的方法：removeCategory(String category)

  • 一个Activity可以包含多个category，只有当Intent包含的category是Activity包含的category的子集时，Activity才能被启动
  • 所有Intent对象都会自动添加Intent.CATAGORY_DEFAULT属性，但是<catagory/>标签中不会自动添加android.intent.category.DEFAULT，所以所有<intent-filter>都要手动添加这一行

Data和Type

对应<intent-filter>元素中的<data>元素
设置data：setData(Uri data)，对应<data>元素中的Uri相关属性，比如scheme、host、type
设置type：setType(String type)，对应<data>元素中的mineType属性

setData()可以只设置data，setType()可以只设置type，但要既设置data又设置type只能使用setDataAndType()

一个Intent最多只能包含一个Data和一个Type，intent中的data和type都已intent-filter中的某一个data元素匹配时，就算匹配成功

Extra

用putExtra和getXXXExtra来传递数据，一个Intent可以包含很多Extra

Extra和Data的区别我想应该是：Extra包含的是数据本身，传过去一个字符串，被接收后，对方就收到了一个字符串；而Data包含的是Uri，是数据的地址，所以对方只是收到了地址，还需要根据地址取出对象

Intent有一些标准Extra，这些应该是用在putExtra方法中的key（键），但是它也规定了需要传递的值的类型（非强制）。用统一的键名和数据类型来传递一些固定类型的数据有利于统一编程规范，但是我还不了解这些标准Extra的意义。

Flag

给intent设置flag：setFlags (int flags)同时设置多个flag，用|隔开

FLAG_ACTIVITY_NEW_TASK

  1. 用这个旗标去启动的activity如果没有设置taskAffinity，那这个旗标就不起作用
  2. 如果设置了taskAffinity，并且这个task不存在，那他就会新建这个task，并且把这个启动的activity作为这个task的根task
  3. 如果设置了taskAffinity，并且这个task已经存在，且这个activity已经作为这个task的根activity存在，那就直接把这个task放到前台
  4. 如果设置了taskAffinity，并且这个task已经存在，但是这个activity并不是这个task的根activity，那把这个task带到前台，并且在这个的task的栈顶再新建这个activity

FLAG_ACTIVITY_SINGLE_TOP

等同于launchMode的singletop属性

FLAG_ACTIVITY_CLEAR_TOP

这个属性和launchMode的singletask有点像，同样是清空目标activity之上的所有activity，但不同的是这个旗标还多一个功能，就是在关闭目标activity之上的activity之前，把新的intent传递给目标activity，这意味着目标activity要被销毁并重建

FLAG_ACTIVITY_CLEAR_TASK

这个旗标的作用是要启动目标activity前，先将这个activity要进入的task清空，然后再新建这个activity。
这个旗标只能和FLAG_ACTIVITY_NEW_TASK一起用，因为task被清空后，这个task就不存在了，所以要new task

FLAG_ACTIVITY_EXCLUDE_FROM_RECENTS

  1. 这个旗标没看懂，API文档的说法是设置了这个旗标的activity不会出现在近期任务列表中，但是近期任务列表本来就是不用来显示activity的，而是用来显示task的
  2. 但有一点可以确定，如果一个task的根activity被设置了这个旗标，那这个task不会出现在近期任务列表中
  3. 这个旗标的作用和<actvity>标签中的android:excludeFromRecents="true"的效果是一样的

FLAG_ACTIVITY_NO_HISTORY

设置了这个旗标的activity会在下次可见前被销毁。

举个例子，有一个activity有这个旗标，那我按home键，这时候什么都没发生，但是等我用近期任务列表切回到原来的task的时候，哪个activity会被销毁（ondestroy()方法会被调用）

FLAG_ACTIVITY_TASK_ON_HOME

  1. 比如，A->B->C->D，如果在C启动D的时候设置了这个标志，那在D中按Back键则是直接回到桌面，而不是C。
  2. 只有D是在新的task中被创建时（也就是D的launchMode是singleInstance时，或者是给D指定了与C不同的taskAffinity并且加了FLAG_ACTIVITY_NEW_TASK标志时），使用FLAG_ACTIVITY_TASK_ON_HOME标志才会生效

FLAG_ACTIVITY_REORDER_TO_FRONT

假设一个Task由四个Activity组成：A,B,C,D。如果D调用startActivity()来启动Activity B，那么，B会移动到历史stack的顶端，现在的次序变成A,C,D,B。在这期间没有activity被销毁，也不会有创建。

如果FLAG_ACTIVITY_CLEAR_TOP标志也设置的话，那么这个标志将被忽略。

FLAG_ACTIVITY_NO_ANIMATION

  1. 启动activity时，禁用启动动画

FLAG_ACTIVITY_FORWARD_RESULT

  1. O ----startActivityForResult()----> A ----FLAG_ACTIVITY_FORWARD_RESULT----> B
  2. A的答复目标是O，如果A在启动B时使用了这个标志，A就会把答复目标O的信息传递给B，以便B将O作为它的答复目标。此时B调用setResult()时的结果信息都会传递给O，而不会给A。并且此时在A中调用setResult()的内容不会生效。我还没发现使A中setResult()生效的方法。

这个标志不能与startActivityForResult()一起使用
