Linux环境

bash 配置文件

~/.profile：shell 登陆时执行，开启子 shell 时，如果存在~/.bash_profile，该文件不被读取。
~/.bashrc：开启子 shell 时执行。
/etc/profile：对所有用户有效。
/etc/bash.bashrc：对所有用户有效。

软件安装位置

/usr：系统级的目录，apt 安装软件的目录。
/usr/local：用户级的程序目录，目录结构类似于 /usr，用户自己编译的软件的安装目录。
/opt：存放可选的程序（比如火狐测试版），而且建议把所有该程序相关的文件放同一个目录中（比如 firefoxBeta），这样要删除的时候直接删除整个目录即可。

用户管理

相关配置文件

/etc/passwd，这些字段的意义分别是：用户名、密码占位符、UID、GID、注释信息、用户主目录、用户的默认shell。
/etc/shadow，用户真正的密码保存文件，加密的，反正看不了。
/etc/group，这些字段的意义分别是：组名、组密码占位符、GID、除了与当前组同名外的其他的该组的用户 /etc/gshadow。

用户

id || id user：查看当前用户或是指定用户的信息。
useradd -g <group> -G <group> -u <UID> -s <shell> -m -o -d <home> <username>
指定用户所属的默认组，不指定就自动创建一个和用户名同名发组。
指定用户所属的额外的组，同一个用户可以属于多个组。
指定用户UID，默认从1000开始递增。
指定默认shell，不指定默认可能是sh也可能是bash。
自动创建主目录。
允许与现有用户使用相同UID，但这两个用户所属的组很可能是不同的。
指定用户主目录。
最后一个是要创建的用户的名字。

passwd [user]：普通用户只能用来修改自己的密码，root可以修改其他用户的密码。创建用户后一定要创建密码，否则用户无法登陆。

userdel [-r] <username>：删除用户，【同时删除用户主目录】，如果该用户的默认组只有这一个用户，组也会被删除。

usermod <option> <username>：修改用户信息，选项很多和useradd相同，值得注意的是可以锁定/解锁用户 groupadd/groupmod/groupdel：这三个要用的时候查文档吧。

gpasswd <group>：组也是可以设置密码的，这个密码的作用是允许别的组的用户在不修改自己的默认组的情况下，临时设置默认组为该组（就是有组密码的这个组，就像用户要登陆必须要有密码一样，临时修改默认组，这个新的组也一定要有组密码）。

newgrp <group>：临时切换默认组。

资源管理

free [-m]：以MB为单位（默认为KB）显示内存使用状况，-/+ buffers/cache之后的才是真实的内存使用情况。
uptime：查看系统持续运行时间。
netstat：和windows中的netstat差不多，一般使用netstat -anp，windows中用-o来显示占用某个端口的进程的pid，linux用-p

top -d <屏幕刷新时间间隔> -n <刷新次数> -p <pid> -u <用户名> -c [显示进程的完成命令行]

load average

假设有一个每分钟能处理100个进程的单核CPU，它1分钟的平均负载是1.0，就表示这一分钟它正好处理了100个进程。
如果是1.7就表示出了被处理的100个进程，还有70个进程在等待中。
也就是说对于单核CPU，1.0就是满载状态，对于双核CPU，这个值是2.0，以此类推。
对于单核CPU，这个值大于0.7就有问题了，大于5.0基本就是死机了。

进程排序

top默认根据CPU的使用率，降序排列。
进入top后，按b（打开/关闭高亮效果），再按x（打开/关闭列的高亮效果），输入>或是<，移动高亮列。top会以高亮列的数据为标准，降序排列进程。

ps有支持三种选项风格，UNIX的单横杠，BSD的无横杠，GNU长选项的双横杠。不仅仅横杠的区别，参数也是不同的。
另外他们能显示的进程信息也不同，比如UNIX风格不能显示进程使用的CPU和内存的百分比。
ps默认显示当前用户在当前终端执行的命令。
-e：显示所有进程。
-f：full，显示更多的列，能显示进程的更多信息，比如用户、PPID。
-l：long，显示比full还要更多的列，比如进程的运行状态。

lsof

列出打开的文件，因为linux中一切都是文件，所以可以用来查看所有被打开的东西。
-i <Internet address>：被打开的网络地址。这个地址的有严格的形式要求，这里简单几下常用的几种。
:port 当前主机的端口。
@ip:port 某个ip地址的某个端口。

-p <pid>：显示某个进程打开的文件。
lsof <file>：显示打开这个文件的进程是哪个。
