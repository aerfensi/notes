bash 脚本

[TOC]

  • bash 对空格的有无敏感，但对其数量不敏感。该有空格的地方一定要有，但是无所谓有几个
  • 脚本参数：$0脚本名，$1第一个参数，$*所有参数（不包括$0）,$#参数数量（不包括$0）
  • 上一个命令的退出状态：$?，0 表示上一个命令无错误，1表示有错误
  • '\n'（"\n"）在正则表达式中被解释为换行，而在bash中被解释为普通字符 n。echo \n的输出为 n。希望\n表示为换行可以使用$'\n'
  • bash 的分隔符是空白符（包括空格、制表、换行等），临时修改分隔符可以在执行命令前oldIFS=$IFS，IFS=$'\n'，执行命令后IFS=$oldIFS

变量

key=value

  • 变量名的首个字符必须是字母
  • 变量名中不能出现空格，如果需要，可以使用_代替
  • 等号左右不能有空格
  • 更精准地使用变量，可以用${key}代替$key

变量类型

  • 本地变量：key=value，当前shell有效
  • 环境变量：export key=value，当前shell及其子shell有效
  • 局部变量：local key=value，当前代码块有效，只能用在函数里

变量传递

子 shell 只能读取父 shell 的 export 变量，且读取到的是这个变量的副本，而管道开启的子 shell 还能读取非 export 的变量，当然也是副本

接受用户输入

  • read key：读入一行内容（不包括换行符）到key
  • read key1 key2：把用户输入一行内容中的第一个空白符前面的数据赋值给key1，后面的全部赋值给key2。当然，在使用的时候可以增加key的数量
  • read -p "input" key：给用户提示
  • read -t n key：n秒内不输入，这条语句就过了，key不会被赋值
  • read -r key：将读取内容中的反斜杠作为普通字符

参数处理

getopts 是 bash 的一个关键字，可以用来处理参数，只支持短参数形式-a aaaa形式，不支持--a=aaa形式
getopt可以支持长参数形式，不过不建议使用

#!/bin/bash
# a、b、c 依次赋值给 arg
while getopts "a:bc" arg # 选项后面的冒号表示该选项需要参数
do
    case $arg in
         a)
            # $OPTIND 表示参数（没有参数就表示选项）在当前参数列表中的位置
            echo "a's arg:$OPTARG  $OPTIND" #参数存在 $OPTARG 中
            ;;
         b)
            echo "b  $OPTIND"
            ;;
         c)
            echo "c  $OPTIND"
            ;;
         ?)  # 当有不认识的选项的时候 arg 为 ?
        echo "unkonw argument"
    exit 1
    ;;
    esac
done

# 解释 $OPTIND（不常用，随便看看即可）
# 把上述代码保存为 test.sh 文件，执行如下命令
./test.sh -b -a hello -c
# -b 时，$OPTIND 为 2，因 test.sh 是第一个参数，故 -b 是 2

# -c 时 $OPTIND 为 5
