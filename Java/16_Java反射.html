16 Java反射

百科上对于反射的定义如下：

Java反射机制是指在运行状态中，对于任意一个类，都能够知道这个类的所有属性和方法；对于任意一个对象，都能够调用它的任意一个方法和属性；这种动态获取的信息以及动态调用对象的方法的功能称为java语言的反射机制。

问题如下：有一个带泛型参数的类Father，它的子类Son指定了Father类的泛型参数的具体类型是String，现在有一个Son类型的对象，如何通过代码知道他父类的具体泛型参数？

可以看到这段代码中用到了三个类：Class、Type、ParameterizedType。

Class虽然用的少，但是书上看的多，所以比较熟。
Class引用的方法一共有三个。

  1. Class.forName("类名字符串") （注意：类名字符串必须是全称，包名+类名）
  2. 类名.class
  3. 实例对象.getClass()

Class对象存在于Class文件中，单例，所以用上述三种方法多次获得Class对象其实都是指向同一个对象的引用。

Type这个东西从没接触过，这个东西要从Java的泛型擦除讲起。Java的泛型是伪泛型，所有泛型对象在编译后都会擦除泛型信息，比如List<object>和List<String>等类型，在编译后都是List类型的，不会有一个名为List<String>的class文件产生。为了表示泛型的信息，Java加入了Type作为Class和其它几个接口的父接口，这些接口我现在只学习其中一个——ParameterizedType，表示泛型参数的具体类型。

为什么看这个

网友的那个MVP框架中定义了带泛型的Activity、Presenter、Model类型，我只需要定义继承自这三个类的类型（当然还要指定泛型的具体类型）就可以了，在Activity的onCreate方法中，我调用super.onCreate，然后网友的Activity就会自动创建Presenter和Model对象，这样我就不用关心，这些对象的创建，可以直接在Activity中使用Presenter对象，在Presenter中使用Model对象了。
那他是怎么知道要创建的对象的类型呢，当然是用反射确定泛型的类型了。
