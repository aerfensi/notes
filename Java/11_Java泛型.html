<?> 与 <T>的区别

<?>：用于使用
<T>：用于定义
比如声明一个类可以用class Plate<T>{}，而在使用中可以使用Plate<? extends Fruit> plate = new Plate<Apple>(new Apple());

这里解释一下为什么上界通配符只能取不能存
因为现在编译器虽然知道这个盘子里装的一定是【水果】或者【它的子类】，但是并不知道它是个什么盘子
如果它是一个可以装任何水果的盘子，而我现在想存入【苹果】，这是可以的
但是，如果它是一个只能存放【苹果】的盘子，而我现在想存入【水果】，这是错误
就像是我可以把一个子类对象传入到一个接受父类对象作为参数的方法中，但是不可以把一个父类对象传入到一个接受子类对象的作为参数的方法中
简单的说，编译器不知道这个容器的具体类型，所以拒绝向容器中存入对象；但是知道容器里存放的是什么东西，所以可以取出对象。

为什么要有上下界通配符

来看一个上界通配符的应用，看一下Collection类中addAll方法的参数
addAll(Collection<? extends E> c)
为什么不是addAll(Collection<E> c)呢？

举个例子，如果有一个Collection对象的泛型是Object，那addAll希望做的是把另一个Collection<Object> 中的数据都读取并添加进来，这自然是没问题的，但是它还希望能把泛型是Object的子类（比如String）的Collection中的数据都读取并保存起来。如果使用addAll(Collection<E> c)是不行的，因为java中Collection<String>不是Collection<Object>的子类，所以Collection<String>的对象不能作为参数传入到addAll(Collection<Object> c)方法中

这时就要使用上界通配符了，Collection<? extends Object>可以说明三点信息

  1. 虽然它不是Collection<Object>的子类，但是他们都是Collection对象
  2. 存储的数据类型是要求的数据类型的子类
  3. 这些数据只能内读取

泛型的使用

简单的说就是在创建对象的时候需要指定泛型的具体类型，比如：List<String> contacts = new ArrayList<>();。
当然也可以不指定具体的泛型类型。

  1. List list：所有存入list，和从list中取出的对象都会是object类型的，当然取出后可以对他将至类型转换
    换句话说如果定义类时，以及创建对象时都没有指定泛型的具体类型，那该类中所有用到泛型参数的地方，泛型参数都会被视为Object型
  2. List<?> list：所有取出的都是Object类型的，但是存入只能是null

开始分析：

一、这个Adapter在定义的时候已经指定了泛型参数的上边界，所以在继承时不能再去指定上边界，否则编译器会认为这个泛型参数有两个上边界

xxxxxxxxxx
5
1
//这是错的
2

//这是对的
4
//泛型参数的名字随意。虽然定义时用的是VH，这里用T也完全没关系
5
SubClass extends RecyclerView.Adapter<T>

二、确定了父类的写法，现在来确定子类的。首先如果不指定父类泛型参数的具体类型，那子类必须处理这个泛型参数

xxxxxxxxxx
4
1
//这种形式是错误的
2
class Son1 extends Father<T>
3
//这种形式才是正确的
4

从上面可以想到，Java的泛型参数是不能继承的，父类有泛型参数的情况下，子类必须重写一边。连泛型参数都不能继承，那泛型参数的边界就更不能继承了，所以正确的写法是一摸一样写一遍
