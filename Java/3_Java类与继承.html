面向对象编程（OOP）的四大基本特性

抽象：将事物的一些特性提取出来，抽象为一个类或者接口。
数据抽象：表示世界中一类事物的特征,就是对象的属性.比如鸟有翅膀,羽毛等(类的属性)。
过程抽象：表示世界中一类事物的行为,就是对象的行为.比如鸟会飞,会叫(类的方法)。

封装：将数据与方法包裹起来，只能通过特定的方式访问方法与数据。Java中使用访问修饰控制符来实现。

继承：一个新的类可以继承自现有类，通过继承，子类可以复用父类的方法和变量。

多态：同一操作作用于不同的对象，可以有不同的解释，产生不同的执行结果。
父类变量可以指向子类实例，当其指向不同类型的子类实例时，同样的操作可以有不同分结果。
Java的重载也是多态。

final修饰符

重载（overload）与重写（override）

重载（overload）的注意点：

  1. 参数类型与数量相同，但是顺序不同也是允许的。
  2. 方法能够在同一个类中或者在一个子类中被重载。
  3. 无法以返回值类型作为重载函数的区分标准。

重写（override）的注意点

  1. 声明为static的方法不能被重写，但是能够被再次声明。
  2. final方法不能被重写。
  3. 可以使用@override标明这是一个重写方法，也能用来检查是否正确重写了方法。
  4. 如果不能继承一个方法，则不能重写这个方法。

覆盖父类成员变量：可以覆盖父类的任意成员变量，即便是fina或者private都没关系。

static

static修饰符可以用来创建静态方法与静态属性，他们也是可以被继承的，但是不能被重写。
重写是实例方法的一个概念，当子类重写了父类的实例方法后，创建一个子类对象，只能调用子类对象的重写方法，而不能调用父类的原方法，除非重新创建一个父类对象。当然，子类中可以使用super调用父类的原方法。
对于静态方法，因为他是属于类的，可以子类调用父类的静态方法（Son.method）,当子类有相同的静态方法时，Son.method调用的是子类的方法， 此时要调用父类的静态方法可以使用Father.method。
所以静态方法不是被重写，而是被隐藏。
