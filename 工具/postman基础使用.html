postman基础使用

组织request

可以直接通过new tab来新建request。


当要保存request，或是使用new request来新建，则必须为request指定一个collection。collection里面还能再建folder，request也可以放到folder中。

所有发送过的请求都可以在history中找到。

请求

在请求的url中加上参数，会自动同步到query params的表格中，反之亦会同步。
不管是get还是post请求，这个参数都会放到url中。虽然get请求的参数应该放在url中，而post的请求应该放在请求体中，jmeter就是这么做的，但是postman这里的是query params，而不是params，所以都是在url中，当然post请求不需要query params就是了。
【send and download】会在发送请求后将响应保存到本地文件，【save】是保存当前请求。

打开console可以看到完整的HTTP请求和响应报文。

点击【code】（在send按钮下面），可以查看请求报文，居然还能自动翻译成代码！

authorization

原先想学一下这几种身份认证的，现在觉得到时候碰到的时候在学吧。

basic auth

将【用户名:密码】做base64编码，可以在请求头中发现有着一行：
Authorization:"Basic bmFtZTpwYXNzd2Q="

inherit auth from parent

从collection中继承authorization，可以在collection->edit—>authorization中设置。

headers

temporary headers：postman自动添加的，不可修改，可以在headers中添加同名属性，来覆盖temporary headers中的同名属性。
bulk edit：切换为纯文本的编辑界面，不是键值对表格这种友好的界面。
presets：预设值，可以将多个header键值对添加到一个组内，后续直接选择这个组，就可以为headers一次性添加多个键值对了。

body

选择raw的话，可以选择文本的格式json、xml这种，将会在headers中自动添加Content-Type属性。
beautify可以格式化json或xml这种文本。

https

校验服务端证书，如果服务端的证书不是postman信任的CA颁布的，这里就不要开启了。

CA Certificates可能是用来验证服务端证书的CA证书，没用过，关着吧，反正我不验证服务端。
在Client Certificates中可以添加客户端的证书。

  1. Host：在这个host中启用客户端校验
  2. crt file：客户端的公钥
  3. key file：客户端的私钥
  4. pfx file：客户端的密钥库，包含公钥和私钥。添加密钥库就不用添加私钥公钥了
  5. Passphrase：私钥或者是密钥库的密码

变量

postman的变量作用范围是：global->collection->environment->local->data。
在此处编辑global/environment，以及设置当前的environment。

  1. global：对所有请求有效
  2. collection：在collection中设置（在variables或pre-request scripts中），对collection中的请求有效
  3. environment：可以创建多个的environment，选择不同的environment就可以使用不同的variable
  4. local：在请求的pre-request scripts中设置，仅对当前请求有效
  5. data：在pre-request scripts中设置，导入外部的csv、json等文件中的数据

可以使用{{变量名}}来使用变量。
